1. Dashboard Introduction

 

Introduction

The dashboard presents the sensor values in a meaningful form to the user.

 

Required Functions

The functions provided by the dashboard are:

  • Sensor functions
    • Show values as dials or temperature gauges
    • Show trend graph of a certain function
  • Administration functions
    • Log in to the system, and remember user/password combination (cookie)
    • Add a sensor to the system
    • Add users to the system (super users only)

 

Architecture

The dashboard reads its values from the database.

  • The database is shared between several companies using the same backend system, but protected by user/password combinations. The MQTT user and password settings determine which userdata is accessible by the system. These need to specified only once.
  • Users have to specify a unique MQTT code which is unique in TTN and separates their data from other users
