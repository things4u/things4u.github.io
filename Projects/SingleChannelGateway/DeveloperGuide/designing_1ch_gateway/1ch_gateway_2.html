Gateway software

This document describes in more detail the design considerations for a 1-channel gateway software. The TTN server-gateway-node communication is according to the document made by Semtech. Although everybody is free to implement his/her own communication protocol for gateway communication, this document will adhere to the standard set by Semtech.

 

Introduction

Several versions have been designed of the single channel gateway software for the ESP8266. The hardware description and build instructions are found in the Hardware Guide.

 

Basic Operation

The following functions must be performed by the gateway:

  • Forward PUSH messages received from LoRa nodes to the backend
  • Send periodical STAT message to the server with information about the gateway status and statistics
  • Send PULL REQUEST message to the server where it asks for downlink messages to send. These PULL_REQUEST messages also serve as a keep-alive type of message for the server (unlike the STAT messages);
  • Several ACK message to confirm messsage receipt to the server
